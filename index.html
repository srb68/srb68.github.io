Privacy Pulse
code

January 2026

I built this Chrome extension because I got curious about how much tracking actually happens when you browse the web. Turns out, it's a lot more than I expected.

Privacy Pulse runs in the background and intercepts all network requests your browser makes using Chrome's webRequest API. It compares each request against a custom database I built of known trackers — over 100 trackers across 7 categories: advertising networks, fingerprinting scripts, data brokers, analytics services, social media trackers, cryptominers, and malware domains.

The tricky part was making it fast. Every single network request needs to be checked, and a typical page load can have hundreds of requests. I used a hash map structure for O(1) lookups so there's basically zero performance impact on browsing. The extension calculates a real-time "privacy score" for each site and shows live visualizations using Chart.js — you can see exactly who's tracking you and how.

This project taught me a lot about browser internals, async JavaScript, and how the advertising/tracking ecosystem actually works. It's kind of disturbing once you see it.

Culinary Connoisseurs
code

December 2025

A full-stack catering management platform I built for my Internet Applications course. The idea is that a catering company could use this to let customers browse menus, get quotes, and book events online.

The backend is PHP with MySQL — I know it's not the trendiest stack, but it was a good exercise in understanding how web apps work at a fundamental level without frameworks abstracting everything away. I designed the database schema myself with 5 relational tables: users, events, menu items, orders, and messages. Wrote complex JOIN queries for the admin reporting dashboard that shows revenue by event type, popular menu items, booking trends, etc.

One feature I'm proud of is the real-time chatbot. It uses AJAX to send messages without page reloads, so users can ask questions about catering options and get instant responses. It's not AI-powered (just pattern matching), but it handles common questions well.

Security was a big focus. I used prepared statements everywhere to prevent SQL injection, implemented proper password hashing with bcrypt, and added input validation with regex for things like phone numbers and emails. After learning about all the ways web apps can be attacked, I got a bit paranoid about sanitizing inputs.

Student Productivity Hub

I wanted to build something I'd actually use, so I made an all-in-one productivity app for students. It combines five tools I found myself switching between constantly: a task manager, grade calculator, Pomodoro timer, notes app, and analytics dashboard.

The analytics dashboard was fun to build. It aggregates data from all the modules to show things like: tasks completed over time, study hours per day, grade trends across semesters, and productivity patterns (apparently I'm most productive on Tuesdays?).

Station Zero
code

Fall 2025

A first-person survival game set on an abandoned space station, built in Unity for my game development course. You wake up alone on a derelict station with failing life support and have to scavenge resources, repair systems, and figure out what happened to the crew — all while dealing with hostile entities that have taken over.

This was my first serious game development project and it pushed me way outside my comfort zone. Game dev is a completely different beast from web dev. I had to learn about game loops, physics engines, collision detection, state machines for AI, and a bunch of Unity-specific stuff like prefabs and scriptable objects.

The AI system was the hardest part. Enemies patrol the station, respond to sounds, and hunt the player. I implemented a finite state machine with states like Idle, Patrol, Alert, Chase, and Attack. The pathfinding uses Unity's NavMesh system but I added custom behaviors for things like investigating noise and losing sight of the player.

Other features: resource management (oxygen, health, power), an inventory system, interactive terminals with logs that tell the story, environmental hazards like radiation zones and depressurized areas, and a progression system where you unlock new areas by repairing station systems.

Unity C# Game Design AI/State Machines 3D Modeling
Personal Expense Tracker
code

August 2025

A desktop app for tracking personal finances, built with Python and Tkinter. I made this because I wanted to understand where my money was actually going, and the existing apps either had too many features or wanted a subscription for basic stuff.

The app lets you log expenses with categories (food, transport, entertainment, etc.), view spending history, and visualize where your money goes. I used Matplotlib to create interactive charts — pie charts for category breakdown, line graphs for spending over time, bar charts comparing months.

Data saves to JSON files, so everything persists between sessions. I implemented full CRUD operations: add expenses, edit past entries, delete mistakes, search and filter by date range or category. Added input validation throughout because I kept accidentally entering negative numbers or leaving fields blank during testing.

This was a good exercise in building a complete, polished desktop application — not just the core functionality but also the little things like confirmation dialogs, error messages, and a UI that doesn't look like it was made in 1995.
